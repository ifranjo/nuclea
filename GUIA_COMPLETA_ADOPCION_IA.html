1. Filosofia Central

Principio #1: Contexto es TODO

Claude no lee mentes. Sin contexto rico = respuesta generica inutil.

La diferencia entre usuario novato y competente = calidad del contexto que dan.

Expectativas Correctas

La IA es una herramienta, no magia. Tu aportas contexto y verificacion. La IA aporta estructura y posibilidades.

Error mas comun: Esperar que la IA "entienda sin explicar". Los usuarios nuevos creen que la IA puede leer mentes. Realidad: necesita contexto explicito.

1.1 Regla de Oro: Estructura del Contexto

Esta estructura siempre funciona:

Estructura de Contexto Rico
1. CONTEXTO: Quien eres y que situacion tienes 2. TAREA: Que necesitas exactamente 3. FORMATO: Como lo quieres recibido 4. RESTRICCIONES: Que evitar o incluir obligatoriamente
Ejemplo real:

"Soy tecnico de taller (CONTEXTO). Necesito calcular el tiempo de ciclo para una pieza con 3 operaciones (TAREA). Dame el resultado en formato tabla (FORMATO). No uses formulas complicadas, explica paso a paso (RESTRICCIONES)."

1.2 Instalacion de Claude Code

Prerequisitos

  • Node.js instalado (version 18+)
  • Terminal (CMD, PowerShell, o Terminal)

Instalacion

npm install -g @anthropic-ai/claude-code

Verificar instalacion

claude --version

Primera sesion

claude

Escribe: Hola, que puedes hacer?

2. Nivel NOOB

NOOB - Primeros Pasos

0-7 puntos

Objetivo: Aprender a interactuar con Claude Code, leer archivos, y hacer cambios simples.

Duracion estimada: 2-4 semanas

2.1 Comandos Basicos

Comando Que hace Cuando usarlo
/help Ver todos los comandos disponibles Cuando no sepas que hacer
/clear Limpiar la conversacion Para empezar tema nuevo
/exit Salir de Claude Code Al terminar sesion

2.2 Leer y Editar Archivos

Leer archivos

Ejemplos de lectura
lee el archivo package.json muestrame el contenido de src/index.js que hay en config.py?

Pedir explicaciones

Ejemplos de explicacion
explicame que hace este archivo que hace la funcion calcularTotal? para que sirve esta linea?

Editar archivos

Ejemplos de edicion
anade un comentario explicando esta funcion cambia el nombre de la variable x por contador corrige el error de sintaxis en linea 15
Importante: Claude Code te mostrara el cambio ANTES de aplicarlo. Puedes aceptar o rechazar.

3. Nivel BASICO

BASICO - Consolidar Fundamentos

8-15 puntos

Objetivo: Usar skills basicos, configurar CLAUDE.md, ejecutar comandos.

Prerequisito: Dominar todo lo del nivel NOOB

3.1 Skills Basicos

/commit Commits Guiados

En vez de escribir commits manuales, Claude Code:

  1. Ve los cambios (git diff)
  2. Sugiere un mensaje de commit
  3. Tu apruebas o modificas
  4. Hace el commit

/docs Generar Documentacion

Genera documentacion automatica para funciones, clases, o archivos completos.

/docs genera documentacion para la funcion calcular

/brainstorming Ideas Antes de Codigo

SIEMPRE usar antes de implementar features nuevas:

/brainstorming quiero anadir un sistema de login

Claude hace preguntas, explora opciones, propone enfoque.

Ejecutar comandos bash

Ejecutar comandos
ejecuta pytest tests/ ejecuta npm run build corre python main.py

3.2 Configuracion CLAUDE.md

Crea un archivo CLAUDE.md en la raiz de tu proyecto:

CLAUDE.md Basico
# CLAUDE.md ## Preferencias - Lenguaje: Python 3.12 - Estilo: Codigo limpio, sin comentarios obvios - Tests: Usar pytest ## Reglas - No crear archivos nuevos sin preguntar - Explicar cambios antes de hacerlos

Claude Code lee este archivo automaticamente y sigue tus preferencias.

4. Nivel INTERMEDIO

INTERMEDIO - Skills Avanzados

16-22 puntos

Objetivo: Dominar planificacion, refactoring, y debugging estructurado.

Prerequisito: Dominar /commit, /docs, /brainstorming, CLAUDE.md

4.1 Planificacion y Refactoring

/plan Planificar Tareas Complejas

Para tareas grandes, ANTES de tocar codigo:

/plan implementar sistema de autenticacion con JWT

Claude analiza el codebase, identifica archivos afectados, propone plan paso a paso.

Regla: Tareas de mas de 1 hora = usar /plan primero.

/refactor Mejorar Codigo Existente

/refactor este archivo tiene mucha duplicacion
/refactor extrae la logica de validacion a una funcion

/requesting-code-review Pedir Review

Antes de mergear, Claude revisa bugs potenciales, code smells, y sugiere mejoras.

4.2 Debugging Estructurado

/systematic-debugging Debug en 4 Fases

/systematic-debugging el login falla intermitentemente
Fase Objetivo
1. Reproducir Confirmar el bug existe
2. Aislar Encontrar donde ocurre
3. Identificar Root cause
4. Verificar Confirmar el fix funciona

CLAUDE.md Avanzado

CLAUDE.md Intermedio
# CLAUDE.md ## Proyecto - Nombre: Mi App - Stack: Python 3.12, FastAPI, PostgreSQL ## Preferencias Codigo - Type hints obligatorios - Docstrings solo en funciones publicas - Tests con pytest ## Workflow - Siempre /plan antes de features grandes - /commit con mensajes en ingles - No crear archivos sin preguntar ## Anti-patterns - No usar print para debug (usar logging) - No hardcodear credenciales

5.1 Skills Personalizados

Estructura de un Skill

~/.claude/skills/mi-skill/
├── SKILL.md           # Obligatorio
└── references/        # Opcional
    └── ejemplos.md
SKILL.md Profesional
--- name: mi-workflow-ci description: Ejecuta pipeline CI/CD local antes de push. Corre tests, lint, y build. --- # CI Local Workflow ## Activacion - `/mi-workflow-ci` - "verifica antes de push" ## Pasos ### 1. Lint Ejecutar: `npm run lint` o `ruff check .` ### 2. Tests Ejecutar: `npm test` o `pytest -v` ### 3. Build Ejecutar: `npm run build` ### 4. Resumen Mostrar: Lint PASS/FAIL, Tests X/Y, Build PASS/FAIL

5.2 MCPs (Model Context Protocol)

MCPs extienden Claude Code con herramientas externas:

MCP Capacidad
filesystem Acceso a archivos fuera del proyecto
github Issues, PRs, repos
postgres Consultas SQL directas
slack Enviar mensajes
brave-search Busquedas web

Configuracion

Archivo: ~/.claude/settings.json

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-filesystem", "/ruta"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-github"],
      "env": {
        "GITHUB_TOKEN": "tu_token"
      }
    }
  }
}

6. Skills Universales (OBLIGATORIOS)

Estos 3 skills son para TODOS los niveles

Independientemente de tu nivel, debes dominar estos skills.

6.1 JARVIS Research

Investigacion profunda con 5 busquedas encadenadas:

Invocar JARVIS
/jarvis-research mejores practicas autenticacion 2026

Las 5 Busquedas

# Tipo Objetivo
1 Amplia Que existe sobre el tema?
2 Llenar hueco Que me falta saber?
3 Profundizar Detalle de lo mas relevante
4 Practico Como aplica a MI caso?
5 Riesgos Que puede fallar?

6.2 Autothink (COT/VOT/META)

Protocolo de debate interno para decisiones complejas:

Invocar Autothink
/autothink deberia usar React o Vue para este proyecto?

Las 3 Rondas

Ronda Sigla Pregunta
1 COT (Chain of Thought) Cual es el problema REAL? Que se seguro?
2 VOT (Verification) Que opciones tengo? Compara objetivamente.
3 META Cual recomiendas y POR QUE podrias estar equivocado?

Niveles de Confianza

Nivel Significado Accion
HIGH Verificado en codigo Proceder
MED Basado en patrones Verificar antes
LOW Especulacion DEBE verificar

6.3 River Trace

Trazar el flujo de un proceso SIN ejecutar nada:

River Trace
Traza el flujo de [proceso] desde [inicio] hasta [fin]. NO ejecutes nada. Solo lee y sigue el camino paso a paso. En cada paso: que entra? que sale? que podria fallar?

7. Anti-patrones y Errores Comunes

Por Tipo de Perfil

Perfil Tecnico (Taller, Mecanicos)
  • Trampa mental: "Esto no es para mi, es para programadores"
  • Error: Demasiado respeto al output (miedo a iterar)
  • No iteran: Creen que si piden otra vez "molestan"

Respuesta: "No es para programadores, es para cualquiera que trabaje con informacion."

Perfil Oficina (PM, Logistica)
  • Trampa mental: "Tiene que salir de mi" (creatividad propia)
  • Error: No iteran, aceptan primer resultado
  • No preguntan: No piden aclaraciones cuando no entienden

Respuesta: "La IA propone, tu decides. Tu criterio sigue siendo el valor."

Errores Universales

"No funciona" y abandonar

El peor antipatron. Siempre hay solucion si añades contexto.

Respuesta: "Seguro que le falta contexto. Enséñame que le dijiste."

"Es mas rapido hacerlo yo"

Ahora si, pero cuando tengas 10 tareas similares... escala.

Como Enseñar a Iterar

Tecnica: Cuestionar
Por que elegiste eso y no Y?

Forzar a explicar decisiones = detectar errores = mejorar.

8. Checklists de Progreso

Nivel NOOB - Semanas 1-4

  • Abrir Claude Code (claude)
  • Usar /help
  • Leer un archivo
  • Pedir una explicacion
  • Usar /clear y /exit
  • Pedir que edite un archivo
  • Aceptar/rechazar un cambio
  • Usar /qaterminal para evaluarte
  • Probar /jarvis-research
  • Usar /autothink para una decision

Nivel BASICO - Semanas 1-4

  • Usar /commit para 3 commits
  • Crear CLAUDE.md en un proyecto
  • Usar /docs para documentar algo
  • Usar /brainstorming antes de una feature
  • Ejecutar tests desde Claude Code
  • Buscar codigo en el proyecto
  • Re-evaluarte con /qaterminal

Nivel INTERMEDIO - Semanas 1-4

  • Usar /plan para una feature
  • Usar /refactor en codigo existente
  • Usar /systematic-debugging en un bug
  • Pedir code review con /requesting-code-review
  • Expandir tu CLAUDE.md
  • Crear un skill personalizado basico
  • Leer sobre MCPs

Nivel AVANZADO - Meses 1-3

  • Crear 2-3 skills profesionales
  • Configurar al menos 1 MCP
  • Usar /delegate para una tarea
  • Usar /coordinating-terminals en proyecto real
  • Configurar hooks personalizados
  • Contribuir un skill al equipo
  • Dominar /noumenon para arquitecturas
  • Mentorear a alguien de nivel NOOB/BASICO
